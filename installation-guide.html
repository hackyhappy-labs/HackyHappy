🖥️서버 초기 설정

빈 Ubuntu 서버에 SSH 접속 후 기본 환경을 구성합니다.

시스템 업데이트

sudo apt update && sudo apt upgrade -y

필수 패키지 설치

sudo apt install -y curl wget git openssl python3 python3-pip jq

일반 사용자 생성 (root로 접속한 경우)

# root로 접속한 경우에만 실행 (이미 일반 사용자면 건너뛰기)
adduser deploy
usermod -aG sudo deploy
su - deploy

Ollama 설치 (RAG 임베딩용)

Ollama는 연락처 벡터 저장(RAG 임베딩)에 사용됩니다. AI 대화는 Groq API가 처리하므로 서버 자원을 거의 사용하지 않습니다.

# Ollama 자동 설치
curl -fsSL https://ollama.ai/install.sh | sh

# 임베딩 모델 다운로드 (약 300MB)
ollama pull nomic-embed-text

# 설치 확인
ollama list
# nomic-embed-text 가 표시되면 성공

Docker 설치

# Docker 자동 설치 스크립트
curl -fsSL https://get.docker.com | sh

# 현재 사용자를 docker 그룹에 추가
sudo usermod -aG docker $USER

# ⚠️ 중요: 그룹 변경 적용을 위해 재접속
exit
# 다시 SSH 접속

Docker 설치 확인

docker --version
docker compose version
docker ps
# 에러 없이 출력되면 성공

방화벽 설정 (선택)

sudo ufw allow 22    # SSH
sudo ufw allow 80    # HTTP
sudo ufw allow 443   # HTTPS
sudo ufw --force enable

📼 통화 녹음 & PDF 보고서

기본값은 OFF입니다. 채팅에서 켜면 됩니다.

녹음 ON/OFF: 👤 통화 녹음 켜줘 / 👤 통화 녹음 꺼줘
PDF ON/OFF: 👤 PDF 보고서 켜줘 / 👤 PDF 보고서 꺼줘

녹음 동작: 법적 고지 멘트(4개국어) → 통화 녹음 → MP3 저장 → Twilio 원본 삭제 → Telegram 알림
PDF 동작: 통화 종료 3초 후 → 대화 전문 + AI 요약 + 통화 시간 포함 PDF 자동 생성

파일 위치: 녹음 ~/OpenWebUI/twilio-bot/data/recordings/ | PDF ~/OpenWebUI/twilio-bot/data/reports/

.env 기본값 변경 (영구): ENABLE_CALL_RECORDING=true / ENABLE_PDF_REPORT=true

📋 통화 결과 보고 (지인에게 전달)

안부전화 후 결과를 관리자가 아닌 다른 연락처에게 전화+문자로 보고할 수 있습니다.

사용법:
👤 김철수에게 전화하고 홍길동에게 결과 보고해줘
👤 김철수에게 안부 확인하고 나한테 알려줘
👤 김철수에게 회의 참석 여부 확인하고 박영희에게 보고해줘

보고 방식 (자동):
① 통화 종료 35초 후 → 보고 대상에게 음성 전화로 결과 요약 전달
② 통화 종료 40초 후 → 보고 대상에게 SMS 문자로 결과 요약 전달
③ 동시에 Telegram으로 관리자에게 항상 알림 (보고 대상과 무관)

※ "나한테 알려줘"라고 하면 관리자 본인 번호로 음성+SMS가 옵니다.
※ 다른 사람을 지정하면 음성+SMS는 그 사람에게, Telegram 알림은 관리자에게 옵니다.

📞 수신전화 자동 보고

외부에서 전화가 걸려오면 AI가 응대한 후 자동으로 관리자에게 보고합니다.

동작 흐름:
외부 전화 수신 → AI 응대 → 대화 기록 추적 → 마지막 대화 60초 후 통화 종료 판단 → 자동 보고

보고 내용 (자동):
① SMS 문자 → 관리자 번호로 80자 요약 발송
② Telegram 알림 → 발신자 이름/번호 + AI 요약
③ PDF 보고서 → 대화 전문 + AI 요약 (ENABLE_PDF_REPORT=true 시)
④ 대시보드 기록 → 통화 기록에 자동 저장

보고 예시 (Telegram):
📞 [수신전화 보고]
👤 발신자: 김철수 (010-1234-5678)
📊 내용: 환불 절차 문의. AI가 환불 정책을 안내함. 추가 서류 필요.

※ 관리자 전화는 보고 대상에서 제외됩니다. 외부 번호만 추적합니다.
※ 연락처에 등록된 번호면 이름이 표시되고, 미등록 번호면 "알 수 없음 + 전화번호"로 표시됩니다.

🔑 API 키 수동 갱신 (403 에러 시)

브라우저 에이전트에서 "접근 거부" 또는 403 Forbidden 에러가 나오면 API 키가 Valves에 입력되지 않았거나 불일치하는 것입니다. 
# 1. 현재 API 키 확인
grep BROWSER_AGENT_API_KEY ~/OpenWebUI/.env

# 2. 출력된 키를 복사 (= 뒤의 긴 문자열)
# 예: BROWSER_AGENT_API_KEY=61bf5a8e...c806

# 3. OpenWebUI에 입력:
#    Workspace → Tools → AI 브라우저 에이전트
#    → ⚙️ (톱니바퀴) → BROWSER_AGENT_API_KEY
#    → 키 붙여넣기 → 저장

# 4. 키를 새로 생성하려면:
NEW_KEY=$(openssl rand -hex 64)
sed -i "s/BROWSER_AGENT_API_KEY=.*/BROWSER_AGENT_API_KEY=$NEW_KEY/" ~/OpenWebUI/.env
cd ~/OpenWebUI && docker compose restart browser-agent
echo "새 키: $NEW_KEY"
# → 출력된 새 키를 Valves에 다시 입력!

설치 방법 (Oracle Cloud 기준)

Step 1: Oracle Cloud 인스턴스 생성

① Oracle Cloud 가입: cloud.oracle.com → 계정 생성 (신용카드 필요, 과금 없음)
② 로그인 → Compute → Instances → Create Instance
③ Image: Ubuntu 22.04 또는 24.04 선택
④ Shape: Ampere A1 (ARM) → OCPUs: 4, Memory: 24GB (Always Free 범위)
⑤ SSH Key: 공개키 업로드 또는 자동 생성 → 개인키 다운로드 보관
⑥ Create → 인스턴스 생성 완료 (Public IP 확인)

Step 2: 방화벽 포트 열기

Oracle Cloud는 기본적으로 모든 포트가 차단되어 있습니다. 2곳에서 열어야 합니다.

① Security List (Oracle 웹 콘솔):
Networking → VCN → 서브넷 클릭 → Security List → Ingress Rules → Add Rules
Source: 0.0.0.0/0 / Protocol: TCP / Destination Port: 80 → 추가
Source: 0.0.0.0/0 / Protocol: TCP / Destination Port: 443 → 추가

② OS iptables (SSH 접속 후): 
sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 80 -j ACCEPT
sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 443 -j ACCEPT
sudo netfilter-persistent save
Cloudflare Tunnel을 사용하면 이 Step 2 전체가 필요 없습니다. 포트를 열지 않아도 외부 접속이 가능합니다.

Step 3: 스크립트 실행

로컬과 동일한 스크립트를 그대로 실행합니다. API 키만 입력하면 자동 설치됩니다.

Step 4: Cloudflare Tunnel 설정 (선택 — 권장)

스크립트 마지막에 Cloudflare Tunnel 설정 여부를 묻습니다. y를 선택하면:
